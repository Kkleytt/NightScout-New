# Настройка серверной части
### Настройте конфигурационной файл
1. Перейдите в папку api
2. Запустите файл config.json и настройте в нем следующие параметры
    1. token - Установите токен доверия, для безопасности от хакеров
    2. host - Установите Ip-адрес для доступа к API сервера (по умолчанию 0.0.0.0)
    3. port - Установите порт для доступа к API из вне локального компьютера (по умолчанию 8080)
    4. rules - Установите права для доступа к типам запросов (true - разрешен, false - запрещен)
3. Перейдите в папку database
4. Запустите файл config.json и настройет в нем слудующие параметры
    1. host - Установите Ip-адрес для доступа к вашей БД (Базе Данных)
    2. port - Установите Порт для доступа к вашей БД
    3. user - Установите имя пользователя для авторизации в БД
    4. password - Установите пароль пользователя для авторизации в БД
    5. database - Установите название БД для подключения к ней
5. Перейдите в папку parser
6. Запустите файл config.json и настройте в нем следующие параметры
    1. url - УстановитеURL адрес вашего NightScput
    2. token - Установите Token для полного доступа к вашему NightScout
    3. count - Установите количество получаемых записей при запросе (меньше - быстрее, больше - медленне, но записи будут за более долгий период ~ неделя при 1000)
    4. api/url - Установите Ip-адрес для доступа к вашему API-серверу (по умолчанию localhost так как parser и API работают на одной компьютере)
    5. api/port - Установите порт доступа к API-серверу (такой же как и в файле api/config.json)
    6. api/token - Установите токен доверия для подключения к API-серверу (такой же как и в файле api/config.json)

 ### Разверуните виртуальное окружение
 Зайдите в терминал в папке server и введите следующую команду
``` bash
python -m venv venv
```
После успешного создания виртуального окружения необходимо его активировать командой
``` bash
venv/Scripts/activate
```
Скачайте все необходимые для проекта библиотеки, используя команду
``` bash
pip install -r requirements.txt
```

### Запуск исполняемого файла
Для запуска главного файла main.py необходимо использовать аргументы во время запуска
Список аргументов:
1. --parse - Запуск парсинга и сохранения данных (Не работает без аргумента --api)
2. --parseLoop - Запуск парсинга и сохранения данных в цикле (Не работает без аргумента --api)
3. --api - Запуск API-сервера
4. --info - Вывести таблицу всех аргументов и их описание

Пример запуска:
``` bash
python main.py --parse --api  # Запуск парсинга и сохранения данных и API-сервера
python main.py --pasreLoop --api  # Запуск бесконечного парсинга и сохранения данных и API-сервера
python main.py --api  # Запуск API-сервера
```

### Настройка демона запуска
1. Создайте скрипт запуска с помощью команды
``` bash
sudo nano /usr/local/bin/start_diabetes.sh
```
2. Запишите в файл следующий скрипт
``` start_diabetes.sh
#!/bin/bash

# Переменные окружения
USER=kkleytt  # Укажите имя пользователя в Оерационной системе с правами Администратора
WORKING_DIR=/home/kkleytt/apps/Diabetes  # Укажите рабочую директорию проекта
VENV_DIR=/home/kkleytt/apps/Diabetes/venv  # Укажите путь к вашему виртуальному окружению

# Активация виртуального окружения
source $VENV_DIR/bin/activate

# Запуск программы
exec python3 $WORKING_DIR/main.py --parseLoop --api  # Укажите с какими параметры вы хотите запускать проект
```
3. Сохраните файл с помощью сочетания клавиш (Ctrl + O) и выйдите из него с помощью сочетания клавиш (Ctrl + X)
4. Сделайте данный скрипт исполняемым с помощью команды
``` bash
sudo chmod +x /usr/local/bin/start_diabetes.sh
```
5. Создайте файл демона
``` bash
sudo nano /etc/systemd/system/diabetes.service
```
6. Запишите в файл демона следующий скрипт
``` diabetes.service
[Unit]
Description=Diabetes Program Daemon
After=network.target

[Service]
User=kkleytt  # Укажите пользователя Операционной системы с правами Администратора
Group=kkleytt  # Кажите у какой группыы пользователей будет доступ к данному демону
WorkingDirectory=/home/kkleytt/apps/Diabetes  # Укажите рабочую директорию проекта
ExecStart=/usr/local/bin/start_diabetes.sh  # Укажите путь к скрипту запуска, который вы сделали на шаге 1
Restart=always
RestartSec=5
StandardOutput=syslog
StandardError=syslog
SyslogIdentifier=diabetes

[Install]
WantedBy=multi-user.target
```
7. Сохраните файл с помощью сочетания клавиш (Ctrl + O) и выйдите из него с помощью сочетания клавиш (Ctrl + X)
8. Убедитесь что файл имеет нужные права доступа
``` bash
sudo chmod 644 /etc/systemd/system/diabetes.service
```
9. Перезагрузите модуль управления демонами в ОС
``` bash
sudo systemctl daemon-reload
```
10. Запустите демона который вы создали
``` bash
sudo systemctl start diabetes
```
11. Проверьте все ли корреткно запустилось
``` bash
sudo systemctl status diabetes
```
12. Включите автозапуск демона
``` bash
sudo systemctl enable diabetes
```

##### Полезные команды для управления демоном
1. Запуск демона
``` bash
sudo systemctl start diabetes
```
2. Остановка демона
``` bash
sudo systemctl stop diabetes
```
3. Перезапуск демона
``` bash
sudo systemctl restart diabetes
```
4. Проверка состояния демона
``` bash
sudo systemctl status diabetes
```
5. Просмотр журнала демона
``` bash
sudo journalctl -u diabetes
```
